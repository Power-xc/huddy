# 클로드 비용 최적화 스킬 (Claude Cost Optimizer Skill)

> **담당:** 비용관리부
> **Tier:** 1
> **Skill ID:** `claude-optimize`
> **출처:** Affaan 직접 측정 권장 세팅

## Purpose

Claude Code의 `~/.claude/settings.json` 최적 세팅을 적용하여
기본 설정 대비 AI API 비용을 최대 80% 절감한다.

## Trigger

- `@claude optimize` — 현재 세팅 점검 후 최적화 가이드 출력
- `@claude optimize apply` — settings.json 자동 적용

## 권장 settings.json

```json
{
  "model": "sonnet",
  "env": {
    "MAX_THINKING_TOKENS": "10000",
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50",
    "CLAUDE_CODE_SUBAGENT_MODEL": "haiku"
  }
}
```

**파일 경로:** `~/.claude/settings.json`

## 절감 효과

| 설정 | 기본값 | 권장값 | 효과 |
|------|--------|--------|------|
| `model` | opus | **sonnet** | ~60% 비용 절감 (코딩 작업 80%는 sonnet으로 충분) |
| `MAX_THINKING_TOKENS` | 31,999 | **10,000** | 숨은 thinking 비용 ~70% 절감 |
| `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` | 95 | **50** | 긴 세션 품질 개선 — 더 일찍 압축해 컨텍스트 품질 유지 |
| `CLAUDE_CODE_SUBAGENT_MODEL` | (상위 모델) | **haiku** | 서브 에이전트 비용 ~80% 절감 |

→ **전체 합산 기본 대비 최대 80% 비용 절감**

## Process

### `@claude optimize` (점검 모드)

```
1. ~/.claude/settings.json 현재 내용 읽기
2. 권장 세팅과 비교
3. 누락·비최적 항목 목록 출력
4. 적용 여부 확인 요청
```

### `@claude optimize apply` (자동 적용 모드)

```
1. ~/.claude/settings.json 현재 내용 읽기
2. 권장 3개 env 항목 + model 병합 (기존 설정 보존)
3. 파일 저장
4. 변경 전/후 diff 출력
```

> **주의:** 기존 커스텀 설정은 덮어쓰지 않고 병합한다. `[ENG]`

## Constraints

- `.env` 파일이 아닌 `~/.claude/settings.json`만 수정 `[SEC]`
- 적용 전 반드시 현재 파일 백업 (같은 디렉토리에 `settings.json.bak`) `[ENG]`
- `model` 변경은 현재 세션에서 즉시 반영되지 않음 — 새 세션부터 적용 `[ENG]`
