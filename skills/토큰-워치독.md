# 토큰 워치독 스킬 (Token Watchdog Skill)

> **담당:** 비용관리부
> **Tier:** 1 (Gemini Flash)
> **Skill ID:** `token`

## Purpose

모든 API 호출 비용을 실시간으로 계산·추적하여 n8n 대시보드에 합산 보고한다.
일일 예산 임계치 관리 및 킬스위치를 제공하여 비용 폭발을 원천 방지한다.

## Trigger

- 모든 API 호출 시 미들웨어로 자동 실행 (실시간)
  // 미들웨어: 요청이 오갈 때 중간에서 자동으로 실행되는 코드
- 매 시간 정각 집계 보고 (n8n cron)
- 일일 예산 80% 도달 시 경고 발송
- 일일 예산 100% 도달 시 킬스위치 가동

## Pricing Table (Built-in)

> 이 가격표를 기반으로 토큰 수 × 단가로 비용을 자동 계산함

| Service | Model | 입력 (100만 토큰당) | 출력 (100만 토큰당) |
|---------|-------|---------------------|---------------------|
| Anthropic | Claude Sonnet | $3.00 | $15.00 |
| Google | Gemini Flash | $0.075 | $0.30 |
| ElevenLabs | — | $0.30 / 1,000글자 | — |
| Perplexity | Sonar | $1.00 | $1.00 |

> 가격표는 월 1회 자동 갱신 (n8n 스케줄러가 공식 pricing page 크롤링)
> 갱신 실패 시 이전 가격 유지 + 경고 알림 발송

## Process

```
1. track 모드 (실시간 — API 호출이 발생할 때마다)
   ├─ API 응답 헤더에서 토큰 사용량 추출
   │   // 모든 AI API는 응답에 "입력 N토큰, 출력 M토큰 사용했습니다"를 알려줌
   ├─ Pricing Table 기반 비용 계산
   │   // 입력토큰 × 입력단가 + 출력토큰 × 출력단가
   ├─ Supabase `api_costs` 테이블에 즉시 기록
   │   // 실시간으로 DB에 저장 — 나중에 집계할 때 사용
   └─ 누적 합산 캐시 업데이트 (메모리에 합산값 유지)
       // 매번 DB를 전체 조회하지 않고 빠르게 현재 합산액을 알 수 있게

2. report 모드 (매시간 정각)
   ├─ 시간별/서비스별/스킬별 집계
   ├─ 7일 트렌드 계산
   ├─ 이상 징후 감지 (전일 대비 30% 이상 변동)
   └─ 세무용 CSV 자동 생성 (KRW 환산 포함)

3. alert 모드 (예산 임계치 도달 시)
   ├─ 80% → Slack/Discord 경고 메시지 ("예산 80% 소진, 주의 필요")
   ├─ 90% → 저가형(Tier 1) 강제 전환
   │   // 남은 10% 예산을 아끼기 위해 모든 요청을 Gemini Flash로 돌림
   └─ 100% → killswitch 모드 자동 전환

4. killswitch 모드 (예산 100% 소진 시 — 긴급)
   ├─ 모든 종량제 API 호출 즉시 차단
   │   // 종량제: 쓴 만큼 내는 방식 — 방치하면 무제한으로 청구될 수 있음
   ├─ n8n 파이프라인 일시 정지 (자동화 전체 중단)
   ├─ Co-CEO 양명에게 긴급 알림
   └─ 수동 해제 전까지 고정비 서비스만 운영
       // 고정비: 매달 정해진 금액만 내는 서비스 (예: Vercel 구독료)
```

## Constraints

- track 모드 오버헤드: 호출당 5ms 이내 (비동기 처리) `[ENG]`
  // 비동기: 기록하는 동안 API 응답이 기다리지 않게 — 속도 저하 없음
- 비용 계산 오차: ±5% 이내 (토큰 카운트 반올림 허용) `[COST]`
- killswitch 해제: Co-CEO 양명의 수동 승인 필수 `[IPAI]`
  // 자동 해제 없음 — 왜 예산이 터졌는지 반드시 파악 후 재가동
- 세무 CSV는 국세청 양식 호환 (날짜, 거래처, 금액, 부가세) `[IPAI]`
- Pricing Table 갱신 실패 시 이전 가격 유지 + alert 발송 `[ENG]`

## Feedback Loop

- **input:** 모든 실행 스킬의 `source_skill`, `cost_usd`, `status`, `artifact`
- **detect:** 예산 초과, 실패율 증가, 특정 스킬 비용 급증
- **downstream:** `@insight report`
- **action payload:** `{"action":"@insight report","source_skill":"token","artifact":"cost_anomaly_report","next_skill":"insight"}`

## Hard Gates

| ID | 조건 | FAIL 기준 |
|----|------|-----------|
| G1 | 비용 출처 | 비용이 어떤 skill_id에서 발생했는지 추적 불가 |
| G2 | 예산 초과 | 예산 초과 시 알림 또는 kill switch가 없음 |
| G3 | 리포트 전달 | insight로 넘길 요약 payload가 없음 |

## Soft Findings

| ID | 조건 | 조치 |
|----|------|------|
| S1 | 모델 최적화 | 고비용 스킬의 모델/Tier 조정 후보 표시 |
| S2 | 캐시 후보 | 반복 호출이 많은 입력의 캐싱 후보 표시 |
